# 模块状态

## 这个页面解决什么

模块状态用于跟踪整个文档站的覆盖范围和成熟度。后续继续扩展时，先看这个页面，再决定补内容、补问题库还是新增技术方向。

## 状态说明

| 状态 | 含义 |
| --- | --- |
| planned | 已规划，尚未创建 |
| started | 已开始，有基础内容 |
| first-version | 第一版可学习，有导览和主要章节 |
| stable | 内容较完整，可作为长期入口 |
| maintenance | 进入维护和问题补充阶段 |
| deprecated | 已过时，保留迁移说明 |

## 优先级判断

后续补内容按这个顺序判断：

1. 用户正在学习或使用的方向优先。
2. 已有模块补齐到 stable 优先于开太多新坑。
3. 真实项目问题优先于概念堆叠。
4. 会影响全站结构的变更要先更新治理文档。
5. 新技术模块必须接入技术库和侧边栏。

## 完成度目标

长期目标不是所有模块文档数一样，而是每个模块都能回答：

- 这个技术解决什么问题。
- 适合谁学。
- 从哪里开始。
- 项目里怎么用。
- 常见问题怎么排查。
- 下一步学什么。

## 更新规则

每次新增模块或明显扩展模块后，同步更新：

- 本页模块状态。
- [技术库总览](/technologies/)。
- [内容扩展路线](/technologies/expansion-plan)。
- 对应侧边栏配置。

然后运行：

```bash
npm run docs:check
npm run docs:build
```
