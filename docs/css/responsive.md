# 响应式设计

## 适合谁看

适合需要让页面在桌面、平板和手机上都可读可操作的学习者。

响应式不是把桌面页面缩小，而是重新安排信息优先级。

## 常见断点

项目可以按实际设计调整，常见参考：

| 宽度 | 场景 |
| --- | --- |
| `1200px+` | 宽桌面 |
| `900px` | 平板或窄桌面 |
| `640px` | 手机横竖屏分界 |
| `390px` | 常见手机宽度 |

不要过度依赖设备型号，优先看内容在什么宽度开始挤压。

## 移动端信息优先级

后台页面移动端应优先显示当前页面核心内容。

不推荐：

```text
整块桌面侧边栏
↓
顶部工具栏
↓
正文内容
```

推荐：

```text
紧凑顶部栏
↓
当前页面标题和核心操作
↓
正文内容
↓
抽屉菜单
```

## 工具栏响应式

```css
.toolbar {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 16px;
}

@media (max-width: 640px) {
  .toolbar {
    align-items: stretch;
    flex-direction: column;
  }
}
```

## 表格响应式

后台表格移动端有三种方式：

| 方式 | 适合场景 |
| --- | --- |
| 横向滚动 | 列较多但必须完整展示 |
| 卡片列表 | 移动端只展示关键字段 |
| 隐藏次要列 | 次要信息可折叠 |

不要让表格硬挤，操作列会很容易变形。

## 实际项目常见问题

### 1. 移动端横向滚动

**排查**

- 是否有固定宽度。
- 是否有 `100vw`。
- 表格或代码块是否超宽。
- Grid 子项是否缺少 `minmax(0, 1fr)`。

### 2. 首屏全是导航

**原因**

桌面侧边栏在移动端直接堆叠。

**解决方案**

移动端使用抽屉、顶部紧凑菜单或底部导航。

### 3. 按钮文字换行很丑

**解决方案**

移动端可以：

- 使用图标按钮。
- 缩短文案。
- 将次要操作放入更多菜单。
- 保持按钮最小宽度。

## 最佳实践

- 以内容是否可读作为断点依据。
- 移动端优先显示核心内容。
- 桌面侧边栏移动端改抽屉或紧凑菜单。
- 表格移动端单独设计，不硬压。
- 每次布局变更都检查 390px 左右视口。
