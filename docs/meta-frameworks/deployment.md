# 部署、缓存与运行时

## 适合谁看

适合已经能本地运行 Nuxt 或 Next，但准备上线到 Node 服务、Docker、静态托管、Serverless、Edge 或企业服务器的学习者。

元框架部署比普通 SPA 更复杂，因为它可能同时包含静态资源、服务端渲染、接口路由、缓存、环境变量和运行时平台差异。

## 先判断渲染和部署形态

上线前先回答：

```text
这个项目是纯静态？
需要 Node server 吗？
需要接口路由吗？
需要服务端读取数据库或私密 API 吗？
是否依赖 Edge runtime？
是否有用户登录态？
```

不同答案对应不同部署方式。

## 常见部署方式

| 方式 | 适合场景 | 注意 |
| --- | --- | --- |
| 静态托管 | 官网、博客、文档、内容更新不频繁 | 动态接口要独立部署 |
| Node server | 需要 SSR、接口、完整框架能力 | 要管理进程、日志、健康检查 |
| Docker | 企业环境、统一交付、可复制部署 | 镜像体积和运行端口要清楚 |
| Serverless | 流量波动、按需执行 | 冷启动和平台限制 |
| Edge | 低延迟、边缘计算 | Node API 支持有限 |

不要在不了解项目能力的情况下直接选择静态导出。静态导出通常会限制服务端能力。

## 静态部署

适合：

- 官网。
- 博客。
- 文档。
- 活动页。
- 内容更新频率较低的页面。

关键检查：

- 所有路由是否能预生成。
- 动态路由参数是否可枚举。
- 页面是否依赖登录态。
- 是否使用 server api。
- 部署路径和资源 base 是否正确。

静态部署后，运行时没有 Node server。如果页面依赖服务端接口能力，需要把 API 独立部署。

## Node server 部署

适合：

- SSR 页面。
- 登录后动态页面。
- BFF。
- 需要读取私密环境变量。
- 需要完整框架能力。

部署检查：

```text
build 命令
start 命令
监听 host 和 port
环境变量
日志位置
健康检查
进程管理
回滚方式
```

Node 服务建议提供健康检查：

```text
GET /health
```

容器里通常监听：

```text
0.0.0.0
```

## Docker 部署

Dockerfile 要区分构建阶段和运行阶段。

通用思路：

```text
安装依赖
↓
构建应用
↓
复制运行所需文件
↓
启动 Node server
```

注意：

- 不把 `.env` 直接打进镜像。
- 不把开发依赖带进运行镜像。
- 端口写清楚。
- 容器内和宿主机端口不要混淆。
- 日志输出到 stdout/stderr，方便平台收集。

## 环境变量

元框架项目必须区分：

| 类型 | 说明 |
| --- | --- |
| 服务端私有变量 | 数据库连接、私密 API key、JWT secret |
| 客户端公开变量 | 公开 API 前缀、站点名称、公开埋点 ID |
| 构建时变量 | build 时已经写入产物 |
| 运行时变量 | 服务启动后读取 |

常见事故：

- 把私钥放进客户端公开变量。
- 构建后才改环境变量，但静态产物不生效。
- 测试环境构建包被发到生产。

## 缓存策略

元框架缓存通常有多层：

```text
浏览器缓存
CDN 缓存
框架缓存
API 缓存
数据库或 Redis 缓存
```

排查缓存问题时要逐层确认。

常见建议：

| 内容 | 建议 |
| --- | --- |
| hash 静态资源 | 长缓存 |
| HTML 入口 | 谨慎缓存 |
| 用户个人数据 | 不公共缓存 |
| 内容详情页 | 可按业务设置 revalidate |
| API 响应 | 明确 Cache-Control |

## 发布和回滚

发布前确认：

- 构建命令通过。
- 目标平台支持当前渲染方式。
- 环境变量完整。
- 关键路由可访问。
- 用户态页面不被错误缓存。
- 静态资源和入口文件版本匹配。
- 回滚包存在。

回滚时注意：

- 数据库迁移是否兼容旧代码。
- CDN 是否需要清理。
- 静态资源旧版本是否还存在。
- 框架缓存是否需要失效。

## 实际项目常见问题

### 问题 1：本地 SSR 正常，静态部署后接口 404

原因是静态部署没有 Node server，server api 不存在。

处理：

- 改用 Node server 部署。
- 或将 API 独立部署为后端服务。
- 或确认项目只使用纯静态能力。

### 问题 2：用户看到旧页面

常见原因：

- HTML 被 CDN 强缓存。
- 新入口引用了旧资源。
- 框架缓存或页面缓存没有失效。

处理：

- 检查 Cache-Control。
- 发布时先上传资源，再切入口。
- 保留旧资源一段时间。
- 必要时清理 CDN。

### 问题 3：生产环境读取不到变量

常见原因：

- 构建时变量和运行时变量混淆。
- 环境变量名称不符合框架公开规则。
- 容器启动时没有注入变量。

处理：

- 启动时打印非敏感配置摘要。
- README 写清变量来源。
- 不在客户端输出私密变量。
