# 项目阶段任务

## 适合谁看

适合准备做 Vue Admin、全栈后台、内容站或 AI 工程项目，但不知道如何拆阶段的人。

项目实战不是“一口气做完整系统”。更可靠的方式是按阶段推进，每个阶段都有明确产出、验收方式和可复盘问题。

## 阶段 1：需求和边界

目标：先确定项目要解决什么问题，不急着写代码。

任务：

1. 写清项目目标。
2. 列出核心用户。
3. 列出核心流程。
4. 确定第一版必须做的功能。
5. 明确暂不做的功能。
6. 写出验收标准。

## 阶段 2：页面和路由

目标：先把前端骨架搭起来，确认信息架构和用户路径。

任务：

1. 建立基础布局。
2. 配置登录页、首页、列表页、详情页、404。
3. 写路由表。
4. 定义菜单结构。
5. 写页面空状态和加载状态。
6. 用 mock 数据跑通页面。

验收：

- 路由层级清晰。
- 刷新页面不白屏。
- 移动端没有横向溢出。
- 页面标题、空状态、错误状态清楚。
- 不依赖真实后端也能演示主流程。

## 阶段 3：接口和数据模型

目标：从 mock 切到真实 API 和数据库。

任务：

1. 定义接口契约。
2. 设计数据表。
3. 写迁移和字段注释。
4. 实现接口。
5. 封装前端请求。
6. 联调成功路径和失败路径。

验收：

- 接口字段和前端类型一致。
- 参数错误返回明确提示。
- 401 和 403 区分。
- 数据库约束能兜底关键规则。
- 联调样例保存到文档或接口工具里。

## 阶段 4：权限和安全

目标：让项目从“能用”变成“不会轻易越权”。

任务：

1. 设计用户、角色、权限关系。
2. 定义权限码。
3. 生成动态菜单。
4. 控制按钮权限。
5. 后端接口做授权校验。
6. 高风险操作写审计日志。

验收：

- 未登录进入登录页。
- 无权限菜单不显示。
- 直接请求无权限接口返回 403。
- 权限码集中维护。
- 权限变更后刷新页面状态正确。

## 阶段 5：工程化和测试

目标：让项目可协作、可构建、可回归。

任务：

1. 配置 lint、format、typecheck。
2. 配置环境变量。
3. 写关键组件或 service 测试。
4. 跑生产构建。
5. 分析包体积。
6. 写 README。

验收：

- 新人能按 README 启动项目。
- CI 至少跑检查和构建。
- 核心逻辑有测试。
- 构建产物能用 HTTP 服务预览。
- 大依赖有解释和懒加载策略。

## 阶段 6：部署和交付

目标：项目能在稳定环境运行，并能回滚。

任务：

1. 配置 Nginx 或静态托管。
2. 配置 API 代理。
3. 配置 Docker 或部署脚本。
4. 写发布检查清单。
5. 写回滚步骤。
6. 上线后观察错误和日志。

验收：

- 首页、二级路由、详情页刷新正常。
- API 请求路径正确。
- 生产环境变量正确。
- 有版本记录。
- 有回滚方案。
- 上线后有健康检查和错误观察。

## 阶段 7：问题复盘和沉淀

目标：把项目中的问题变成可复用知识。

任务：

1. 记录联调问题。
2. 记录上线问题。
3. 记录权限问题。
4. 记录性能问题。
5. 做故障复盘。
6. 更新项目文档和问题库。

验收：

- 每个问题有现象、原因、解决方案和预防方式。
- 复盘不追责，关注系统改进。
- 检查清单能根据复盘更新。
- 问题能沉淀到对应文档模块。
