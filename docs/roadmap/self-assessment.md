# 能力自测

## 适合谁看

适合学了一段时间后，不确定自己到底是“看懂了”还是“能做了”的人。

能力自测的原则是：不要问“我学完了吗”，而要问“我能否独立完成可验证任务，并解释关键原因”。

## 评分方式

每项能力按 0 到 3 分评估：

| 分数 | 含义 |
| --- | --- |
| 0 | 完全不会，或只能照抄 |
| 1 | 能在参考资料帮助下完成 |
| 2 | 能独立完成常规场景 |
| 3 | 能处理异常场景，并能解释取舍 |

建议进入下一阶段的最低标准：

- 核心项至少 2 分。
- 项目交付项至少 2 分。
- 排错项至少 1 分。
- 不能有关键安全项 0 分。

## 前端基础自测

| 能力 | 自测任务 |
| --- | --- |
| HTML 语义 | 写一个文章详情页，正确使用标题、段落、列表、链接和图片 |
| CSS 布局 | 用 Flex 和 Grid 分别实现一个响应式卡片列表 |
| 响应式 | 在 390px 和 1440px 下页面都可读可操作 |
| JavaScript | 实现列表筛选、排序和分页 |
| 浏览器 | 能解释一次页面加载涉及哪些资源和请求 |
| 调试 | 能用 DevTools 定位控制台错误、网络错误和布局问题 |

通过标准：

- 页面没有明显重叠和溢出。
- 交互逻辑不是硬编码。
- 能解释代码为什么这样写。

## Vue 前端自测

| 能力 | 自测任务 |
| --- | --- |
| 组件设计 | 拆出搜索栏、表格、表单弹窗和状态展示组件 |
| 组合式 API | 把列表查询逻辑封装成 composable |
| 路由 | 实现登录页、列表页、详情页和 404 |
| Pinia | 保存用户信息、权限和跨页面状态 |
| 请求封装 | 统一处理 loading、错误、401 和 403 |
| 权限 | 根据服务端权限生成菜单和按钮 |
| 表单 | 实现编辑表单校验、提交和错误回显 |
| 测试 | 给关键 composable 或组件写测试 |

通过标准：

- 组件不直接承担所有业务。
- 权限不是只靠前端隐藏。
- 请求错误有统一处理。
- 页面刷新后状态恢复合理。

## Node 后端自测

| 能力 | 自测任务 |
| --- | --- |
| API 设计 | 设计用户列表、详情、新增、编辑和删除接口 |
| 参数校验 | 对 body、query、params 做明确校验 |
| 鉴权 | 未登录返回 401 |
| 授权 | 无权限返回 403 |
| 分层 | controller、service、repository 职责清楚 |
| 数据库 | 设计用户、角色、权限表 |
| 事务 | 用户创建和角色绑定同成功同失败 |
| 日志 | 每个请求能通过 request id 追踪 |
| 测试 | 覆盖成功、参数错误、未登录、无权限和资源不存在 |

通过标准：

- 不在路由里直接堆业务和 SQL。
- 数据库约束能兜底关键规则。
- 错误响应结构统一。
- 关键安全边界在服务端。

## 数据库自测

| 能力 | 自测任务 |
| --- | --- |
| 建模 | 根据用户权限业务设计表结构 |
| 注释 | 为表、字段、索引、约束写业务说明 |
| 索引 | 为列表查询和关联查询设计索引 |
| 事务 | 说明哪些操作必须放事务里 |
| 迁移 | 写出新增字段的安全迁移步骤 |
| ORM | 用 repository 封装查询，不让 controller 直接调用 ORM |
| 备份 | 说明迁移前如何备份和恢复验证 |
| 安全 | 标记敏感字段并设计脱敏规则 |

通过标准：

- 能解释每个字段为什么存在。
- 索引不是凭感觉添加。
- 迁移考虑旧数据和回滚风险。

## 工程化与部署自测

| 能力 | 自测任务 |
| --- | --- |
| Vite | 配置 alias、env、proxy 和 build base |
| 规范 | 配置 lint、format、typecheck |
| 依赖 | 能解释 dependencies 和 devDependencies 区别 |
| 测试 | 区分单元测试、组件测试和 E2E |
| 构建 | 生产构建并用 HTTP 服务预览 |
| 包体积 | 找出最大依赖并说明是否影响首屏 |
| CI | 设计 install、lint、test、build 流程 |
| 部署 | 配置 Nginx history fallback 和 API 代理 |
| 回滚 | 说明如何切回上一个版本 |

通过标准：

- 本地开发和生产部署边界清楚。
- 环境变量不泄漏私密信息。
- 构建、部署、回滚有文档。

## AI 工程自测

| 能力 | 自测任务 |
| --- | --- |
| API 调用 | 封装模型调用、错误、重试和日志 |
| Prompt | 为任务写清角色、输入、输出和边界 |
| 结构化输出 | 对模型输出做 schema 校验 |
| RAG | 文档切分、检索、引用来源和拒答 |
| Agent | 限制工具、步骤、审批和风险边界 |
| 评测 | 建立样例集并跑回归 |
| 成本 | 记录 token、延迟和费用 |
| 安全 | 权限过滤、敏感信息处理和人工兜底 |

通过标准：

- 模型不直接决定权限。
- 答案能说明来源。
- 改 prompt 或模型前后能评测。

```md
## 自测日期

### 当前路线

### 本次自测分数

| 能力 | 分数 | 证据 |
| --- | ---: | --- |

### 最大短板

### 下阶段计划
```

证据可以是代码、截图、部署地址、测试结果、问题复盘或文档说明。
