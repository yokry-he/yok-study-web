# 图解 AI 工程核心概念

## 适合谁看

适合已经会做 Web、Node.js 或后台系统，但想把大语言模型、RAG、工具调用、Agent、评测和上线治理接入真实项目的人。

AI 工程不是“调一次模型接口”。真正上线时，你需要处理输入、权限、提示词版本、检索质量、工具边界、评测、成本、延迟和安全。

## 你会学到什么

- AI 功能从用户输入到模型输出的基本链路。
- Prompt、结构化输出、工具调用、RAG 和 Agent 的关系。
- RAG 为什么不只是向量数据库。
- 工具调用为什么必须有权限和确认。
- 评测如何防止提示词或模型升级带来退化。
- AI 功能上线前要看哪些风险。

## AI 功能调用链路

```mermaid
flowchart TD
  A[用户输入] --> B[业务校验和权限判断]
  B --> C[提示词模板]
  C --> D[模型调用]
  D --> E[输出解析和校验]
  E --> F{是否通过}
  F -- 否 --> G[拒答、重试或人工处理]
  F -- 是 --> H[返回业务结果]
  H --> I[记录日志、成本、耗时]
```

模型输出不能直接等于业务事实。关键结果要经过结构校验、权限校验、来源校验或人工确认。

## Prompt 在系统中的位置

```mermaid
flowchart LR
  A[业务目标] --> B[Prompt 模板]
  B --> C[变量填充]
  C --> D[模型请求]
  D --> E[模型输出]
  E --> F[评测和日志]
  F --> B
```

Prompt 应该像代码一样管理：

- 有版本。
- 有评审。
- 有变更记录。
- 有评测集。
- 能回滚。

不要把 prompt 散落在页面组件里。

## 结构化输出和工具调用

```mermaid
flowchart TD
  A[用户需求] --> B[模型判断输出或工具]
  B --> C{需要工具吗}
  C -- 否 --> D[结构化 JSON 输出]
  C -- 是 --> E[生成工具调用参数]
  E --> F[业务系统校验权限和参数]
  F --> G[执行工具]
  G --> H[工具结果返回模型或用户]
  D --> I[程序解析和校验]
```

工具调用的关键原则：

- 模型可以建议调用工具。
- 业务系统决定是否允许调用。
- 高风险操作需要人工确认。
- 工具参数必须校验。
- 工具结果要记录日志。

## RAG 检索链路

```mermaid
flowchart TD
  A[文档源] --> B[清洗和切分]
  B --> C[生成向量]
  C --> D[写入索引]
  E[用户问题] --> F[查询改写]
  F --> G[向量和关键词检索]
  G --> H[重排和过滤]
  H --> I[组装上下文]
  I --> J[模型回答]
  J --> K[来源引用]
```

RAG 质量不只取决于模型。文档质量、切分方式、检索策略、权限过滤、上下文组装都会影响最终答案。

## Agent 工作流

```mermaid
flowchart TD
  A[任务目标] --> B[规划下一步]
  B --> C{是否需要工具}
  C -- 是 --> D[调用工具]
  D --> E[观察结果]
  E --> F{是否完成}
  C -- 否 --> G[直接回答]
  F -- 否 --> B
  F -- 是 --> H[输出结果]
```

Agent 适合多步骤、不确定路径、需要工具反馈的任务。不要把简单分类、摘要、格式转换都做成 Agent。能用普通 API 调用解决，就不要引入多步自主循环。

## 评测和回归

```mermaid
flowchart LR
  A[评测样例] --> B[当前 Prompt 和模型]
  B --> C[运行输出]
  C --> D[规则评分或人工评分]
  D --> E[对比基线]
  E --> F{是否退化}
  F -- 是 --> G[阻止上线]
  F -- 否 --> H[允许灰度]
```

评测集至少要覆盖：

- 正常问题。
- 边界问题。
- 资料不足问题。
- 权限限制问题。
- 恶意或诱导输入。
- 格式要求严格的问题。

## 上线治理

```mermaid
flowchart TD
  A[AI 功能准备上线] --> B[权限边界]
  A --> C[成本预算]
  A --> D[延迟目标]
  A --> E[日志和追踪]
  A --> F[评测通过]
  A --> G[人工兜底]
  B --> H[灰度发布]
  C --> H
  D --> H
  E --> H
  F --> H
  G --> H
```

AI 功能上线清单：

- 输入和输出是否脱敏。
- 是否越权读取资料。
- 是否有成本上限。
- 是否有超时和重试策略。
- 是否能解释答案来源。
- 是否能拒答。
- 是否有人工兜底。

## AI 工程排错路径

```mermaid
flowchart TD
  A[AI 输出异常] --> B{是否资料不足}
  B -- 是 --> C[检查 RAG 文档和检索结果]
  B -- 否 --> D{是否格式错误}
  D -- 是 --> E[检查 schema 和输出校验]
  D -- 否 --> F{是否越权或危险}
  F -- 是 --> G[检查权限过滤和工具边界]
  F -- 否 --> H[检查 prompt、模型版本和评测]
```

## 实际项目常见问题

### 问题 1：模型回答看起来合理但事实错误

要求答案引用来源；资料不足时拒答；关键结论用业务系统或人工确认。

### 问题 2：RAG 答非所问

不要只改 prompt。先检查文档切分、检索 query、topK、重排、权限过滤和上下文是否包含正确资料。

### 问题 3：工具调用做了危险操作

高风险工具必须由业务系统二次确认。模型不能直接决定删除、付款、审批、授权等动作。
